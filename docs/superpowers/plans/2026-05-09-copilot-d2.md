# Copilot D2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Copilot D2 — VRL 生成技能 + Insert into editor + Library 三頁 page_context + 三顆新 quick-buttons。

**Architecture:** 一份 plan、拆三個 milestone（M1/M2/M3）。M1 = VRL 生成 + Insert（含 store editor bridge、diff dialog、safety banner、per-skill model override）。M2 = Library 三種 page_context（discriminated union schema 重構 + 兩個新 hook）。M3 = 三顆 quick-button + vrl_optimize/anomaly skill prompt + 比對 Library 串 `/api/v1/analyzer/match`。每 milestone 結束開 PR、過了再進下一個。

**Tech Stack:** FastAPI · Anthropic SDK · Pydantic · Zustand 5 · React 19 · Next.js 16（重要：見 `web/AGENTS.md`）· `@uiw/react-codemirror` · MSW · Vitest · pytest-asyncio · biome.

# Skill: vrl_generate

You are generating VRL (Vector Remap Language) parse rules.

## Process (follow in order)
1. Read <logs> + <current_vrl>; identify the format and any existing structure.
2. List the fields to extract. State each field's source position
   (regex group / json path / csv index).
3. Write VRL. Wrap the code in ```vrl ... ``` (exactly one fenced block,
   language tag must be `vrl`). The block is what the user will Insert
   into the editor.
4. After the code block, list edge cases or limitations:
   - what fields might be missing in some logs
   - which engine version this targets (read <vrl_engine> from <facts>)
   - what was intentionally NOT extracted

## You must NOT
- Invent fields not visibly present in <logs>.
- Hard-code API keys, tokens, passwords, hostnames of production systems.
  Use VRL `del()` if a sensitive field needs removal.
- Use VRL syntax that the engine version in <facts><vrl_engine> doesn't
  support (e.g., 0.32 syntax when engine is 0.25).
- Output more than one ```vrl block. If you need to show alternative
  approaches, describe them in prose; pick one canonical version for
  the fenced block.

## Example output structure

這個 log 是 syslog + PAN-OS CSV 結構。我會用 parse_syslog 抓 header，

const VRL_BLOCK_RE = /```vrl\n([\s\S]*?)\n```/;

from xml.sax.saxutils import quoteattr

def _render_library_overview_xml(ctx, *, max_products: int) -> str:
    lines = ['<page_context page="library_overview">']
    lines.append("  <facts>")
    # filters as attribute set; None 值跳過

        f"{k}={quoteattr(v)}" for k, v in (ctx.filters or {}).items() if v is not None
    )
    lines.append(f"    <filters {filter_attrs}/>" if filter_attrs else "    <filters/>")
    lines.append(f"    <vendor_count>{ctx.vendor_count}</vendor_count>")
    lines.append(f"    <product_count>{ctx.product_count}</product_count>")
    lines.append("  </facts>")
    missing = ctx.products_missing_parse_rule or []

    )
    for slug in missing[:max_products]:
        lines.append(f'    <product slug={quoteattr(slug)}/>')
    lines.append("  </products_missing_parse_rule>")
    lines.append("</page_context>")

import { useEffect, useMemo, useRef } from "react";
import { useDebounce } from "use-debounce";

import { useCopilotStore } from "@/lib/copilot/store";

type ProductLite = {
  slug: string;
  isEmpty: boolean;
  logTypeCounts: { total: number; published: number; draft: number };
};
type GroupLite = { vendor: { slug: string }; products: ProductLite[] };

export type LibraryOverviewStateForCopilot = {
  filters: { status?: string | null; q?: string | null };
  groups: GroupLite[];
};

export function useLibraryOverviewCopilotContext(state: LibraryOverviewStateForCopilot): void {
  const setPageContext = useCopilotStore((s) => s.setPageContext);

  const [debouncedFilters] = useDebounce(state.filters, 200);

  const summary = useMemo(() => {
    const vendorCount = state.groups.length;
    const products = state.groups.flatMap((g) => g.products.map((p) => ({ vendor: g.vendor.slug, ...p })));
    const productCount = products.length;
    const missing = products
      .filter((p) => p.isEmpty || p.logTypeCounts.published === 0)
      .map((p) => `${p.vendor}/${p.slug}`);
    return { vendorCount, productCount, missing };
  }, [state.groups]);

  const latestRef = useRef({ summary, filters: debouncedFilters });
  latestRef.current = { summary, filters: debouncedFilters };

  useEffect(() => {
    const { summary, filters } = latestRef.current;
    setPageContext({
      page: "library_overview",
      filters: { status: filters.status ?? null, q: filters.q ?? null },
      vendorCount: summary.vendorCount,
      productCount: summary.productCount,
      productsMissingParseRule: summary.missing,
    });
    return () => setPageContext(null);
  }, [setPageContext,
      debouncedFilters.status, debouncedFilters.q,
      summary.vendorCount, summary.productCount, summary.missing.length]);
}

import { useState } from "react";
import { apiFetch } from "@/lib/api/client";

const [matching, setMatching] = useState(false);

async function handleMatchLibrary() {
  if (matching || !ctx || ctx.page !== "analyzer" || ctx.logs.length === 0) return;
  setMatching(true);
  try {
    const r = await apiFetch<{ data: { candidates: Array<{ vendor_slug: string; product_slug: string; log_type_name: string; confidence: number; reason: string }> }}>(
      "/api/v1/analyzer/match",
      { method: "POST", body: { raw_log: ctx.logs[0], top_k: 3 } },
    );
    const lines = r.data.candidates.map((c, i) =>
      `Candidate ${i + 1}: ${c.vendor_slug}/${c.product_slug}, ${c.log_type_name}, confidence=${c.confidence.toFixed(2)}\n  reason: ${c.reason}`,
    );
